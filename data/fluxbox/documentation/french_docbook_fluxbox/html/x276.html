1. NextWindow / PrevWindow: NextWindow/PrevWindow prend un argument numérique, ce qui est peu difficile à expliquer à l'intérieur du tableau ci-dessus. Voilà comment cela fonctionne.

L'argument entier qui défini les options sont prises dans la table ci dessous et utilisées avec l'opérateur OR :

Valeur du Bit Option
1Naviguer entre les onglets réduits (Traiter les groupes de tabs comme une seule fenêtre)
2Naviguer entre les fenêtres collées
4Naviguer entre les fenêtres ombrées

Aditionnez les nombres ci dessus pour spécifier plusieurs options. Le résultat est le paramètre pour NextWindow/PrevWindow. Vous pouvez prendre ces valeurs dans la table suivante :

ParamètreOptions
0Pas de skipping
1Naviguer entre les onglets réduits
2Naviguer entre les fenêtres collées
3Naviguer entre les onglets réduits/fenêtres collées
4Naviguer entre les fenêtres ombrées
5Naviguer entre les onglets réduits/fenêtres ombrées
6Naviguer entre les fenêtres collées/fenêtres ombrées
7Naviguer entre les onglets réduits/fenêtres collées/fenêtres ombrées

2. HorizontalIncrement / HorizontalDecrement / VerticalIncrement / VerticalDecrement: Je dis "une unité" quand je décris le redimensionnement. C'est en particulier à cause des xterm/aterm/Eterm qui, au lieu d'ajouter un pixel, ajoutent un caractère d'espace.

Les autres programmes devraient être redimensionner d'un pixel.
